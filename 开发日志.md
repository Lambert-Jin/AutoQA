### Phase 1：项目骨架 + VLM Provider

**新增文件：**
- `auto_qa/__init__.py` — 包入口
- `auto_qa/config/__init__.py` / `settings.py` — VLMConfig、AutoGLMConfig、DeviceConfig 数据类，支持 `${VAR}` 环境变量解析
- `auto_qa/asserter/vlm_providers/__init__.py` — VLMProvider 协议 + `create_vlm_provider()` 工厂
- `auto_qa/asserter/vlm_providers/qwen.py` — QwenVLProvider（OpenAI 兼容 API）
- `auto_qa/asserter/vlm_providers/gemini.py` — GeminiProvider（Google GenAI SDK）
- `pyproject.toml` — 项目配置，依赖 openai、google-genai、pyyaml
- `.gitignore` / `.env.example` — 安全管理 API key

**配置调整：**
- VLM 默认改为 Gemini 3 Pro（`provider="gemini"`, `model="gemini-3-pro"`）
- AutoGLM 改为 API 调用（`model="autoglm-phone"`，智谱 BigModel API）
- API key 全部通过环境变量注入，不硬编码

**问题解决：**
- `pip install` 时 tiktoken 编译失败（缺 Rust）→ 升级 pip 后可下载预编译 wheel
- setuptools 报 "Multiple top-level packages"（发现了 auto_qa 和 midscene）→ 在 pyproject.toml 添加 `[tool.setuptools.packages.find] include = ["auto_qa*"]`

**图片格式兼容：**
- 新增 `_utils.py`，通过文件头魔数自动识别 PNG/JPEG/WebP
- QwenVLProvider 和 GeminiProvider 的 mime_type 不再硬编码为 `image/png`

**测试：** `tests/test_phase1.py` — 验证模块导入、工厂分发、Gemini API 调用（遇到免费额度限流 429）

### Phase 2：断言能力

**新增文件：**
- `auto_qa/config/settings.py` — 新增 Screenshot（base64/width/height/timestamp/id）和 AssertResult（passed/reason/confidence/retried）数据类
- `auto_qa/asserter/prompts.py` — ASSERT_SYSTEM_PROMPT，要求 VLM 输出 `{passed, reason, confidence}` JSON
- `auto_qa/asserter/asserter.py` — Asserter 核心类：`verify(screenshot, expectation)` 调 VLM → 解析 JSON → 返回 AssertResult；`_parse_response` 容忍 markdown 代码块包裹，解析失败返回 `passed=False, confidence=0`
- `auto_qa/screenshot/manager.py` — ScreenshotManager：`capture(device)` 从设备截图、`from_file(path)` 加载本地图片、`save()` 持久化；自动解析 PNG/JPEG 宽高

**测试：** `tests/test_phase2.py` — 离线测试（数据类、JSON 解析、ScreenshotManager）+ 在线测试（对截图做正/反两个断言，验证结果相反）。已通过。

### Phase 3：执行能力（TestExecutor）

**新增文件：**
- `auto_qa/executor/__init__.py` — 导出 TestExecutor、ExecutorActionResult
- `auto_qa/executor/executor.py` — TestExecutor 核心实现

**核心设计：**
- 复用 Open-AutoGLM 的 `ModelClient`、`ActionHandler`、`MessageBuilder`、`parse_action`、`get_system_prompt`
- 自管理 `_context` 列表，整个 TestCase 共享上下文（不每步 reset）
- `execute_action(description)` — 注入新指令到已有上下文，内部多轮循环直到 AutoGLM 返回 finish
- `handle_unexpected(instruction, max_steps=3)` — 向已有上下文注入清理指令处理弹窗等
- `reset()` — TestCase 切换时清空上下文
- 每轮执行后通过 `MessageBuilder.remove_images_from_message()` 移除旧图片节省 token

**依赖处理：**
- Open-AutoGLM 通过 `PYTHONPATH` 引入（`sys.path.insert`）
- 补装了 `Pillow` 依赖（Open-AutoGLM 的 iOS 截图模块需要 PIL）

**测试：** `tests/test_phase3.py` — 离线测试（数据类、初始化、reset）+ 在线测试（需要 ADB 设备 + AutoGLM API，验证单步执行、上下文保持、弹窗处理、reset 后新任务）。已通过。

### Phase 4：编排串联（第一个可交付版本）

**新增文件：**
- `auto_qa/suite.py` — 数据模型：ActionStep、AssertStep、TestCase、TestSuite、Timing、StepResult、TestCaseResult、TestSuiteResult
- `auto_qa/planner/__init__.py` / `parser.py` — YAML 解析器：读取 YAML → 解析 device/config/tasks → 构建 TestSuite；支持 `${ENV_VAR}` 环境变量替换
- `auto_qa/runner.py` — TestRunner：`run_suite()` 遍历 TestCase → `run_case()` reset executor + 遍历 steps → `_run_action()` / `_run_assert()` 含容错重试；控制台实时输出每步结果
- `main.py` — CLI 入口：`python main.py run examples/xxx.yaml --device-type adb -v`
- `examples/toutiao_comment.yaml` — 示例测试用例

**修复：**
- `ScreenshotManager.capture()` 适配 Open-AutoGLM 的 `DeviceFactory.get_screenshot()` 返回 Screenshot 对象（有 base64_data/width/height 字段）
- Runner 中使用 `get_device_factory()` 全局函数获取设备工厂

**测试：** `tests/test_phase4.py` — 离线测试（数据模型、YAML 解析、环境变量解析）；端到端测试通过 `python main.py run examples/toutiao_comment.yaml -v` 执行
