### Phase 2：断言能力

**新增文件：**
- `auto_qa/config/settings.py` — 新增 Screenshot（base64/width/height/timestamp/id）和 AssertResult（passed/reason/confidence/retried）数据类
- `auto_qa/asserter/prompts.py` — ASSERT_SYSTEM_PROMPT，要求 VLM 输出 `{passed, reason, confidence}` JSON
- `auto_qa/asserter/asserter.py` — Asserter 核心类：`verify(screenshot, expectation)` 调 VLM → 解析 JSON → 返回 AssertResult；`_parse_response` 容忍 markdown 代码块包裹，解析失败返回 `passed=False, confidence=0`
- `auto_qa/screenshot/manager.py` — ScreenshotManager：`capture(device)` 从设备截图、`from_file(path)` 加载本地图片、`save()` 持久化；自动解析 PNG/JPEG 宽高

**测试：** `tests/test_phase2.py` — 离线测试（数据类、JSON 解析、ScreenshotManager）+ 在线测试（对截图做正/反两个断言，验证结果相反）。已通过。

### Phase 3：执行能力（TestExecutor）

**新增文件：**
- `auto_qa/executor/__init__.py` — 导出 TestExecutor、ExecutorActionResult
- `auto_qa/executor/executor.py` — TestExecutor 核心实现

**核心设计：**
- 复用 Open-AutoGLM 的 `ModelClient`、`ActionHandler`、`MessageBuilder`、`parse_action`、`get_system_prompt`
- 自管理 `_context` 列表，整个 TestCase 共享上下文（不每步 reset）
- `execute_action(description)` — 注入新指令到已有上下文，内部多轮循环直到 AutoGLM 返回 finish
- `handle_unexpected(instruction, max_steps=3)` — 向已有上下文注入清理指令处理弹窗等
- `reset()` — TestCase 切换时清空上下文
- 每轮执行后通过 `MessageBuilder.remove_images_from_message()` 移除旧图片节省 token

**依赖处理：**
- Open-AutoGLM 通过 `PYTHONPATH` 引入（`sys.path.insert`）
- 补装了 `Pillow` 依赖（Open-AutoGLM 的 iOS 截图模块需要 PIL）

**测试：** `tests/test_phase3.py` — 离线测试（数据类、初始化、reset）+ 在线测试（需要 ADB 设备 + AutoGLM API，验证单步执行、上下文保持、弹窗处理、reset 后新任务）。已通过。

### Phase 4：编排串联（第一个可交付版本）

**新增文件：**

- `main.py` — CLI 入口：`python main.py run examples/xxx.yaml --device-type adb -v`
- `examples/toutiao_comment.yaml` — 示例测试用例

### 规划层实现（LLM Planner）

**新增文件：**
- `config/settings.py` — 新增 `PlannerConfig` 数据类，使用 `gemini-3.1-pro-preview` 模型，复用 `${GEMINI_API_KEY}`
- `planner/prompts.py` — `PLANNER_SYSTEM_PROMPT`，指导 LLM 将自然语言分解为 action/assert 步骤，输出 JSON 格式
- `planner/llm_planner.py` — 核心规划逻辑：
  - `plan_test_case(description, planner_config)` — 调用 LLM 将自然语言转换为 TestCase
  - `generate_yaml_content(test_case)` — 将 TestCase 转为完整 YAML 文件内容（含 `${ENV_VAR}` 占位符）
  - `_parse_plan_response()` — 解析 LLM 返回的 JSON，容忍 markdown 代码块包裹
  - 支持 Gemini（google-genai SDK）和 Qwen（OpenAI 兼容 API）两种 provider

**CLI 新增两种模式：**
- `python main.py generate "自然语言描述" -o output.yaml` — 生成 YAML 文件，不执行
- `python main.py interactive --device-type adb` — 交互式模式：输入描述 → 规划预览 → 确认执行 → 循环

**设计决策：**
- 规划器使用独立的 `PlannerConfig`，与 VLM 断言配置分离，模型和参数可独立调优
- 生成的 YAML 复用现有格式，可直接通过 `run` 模式执行
- 交互模式每轮测试后 executor 不 reset，保持上下文连续性
