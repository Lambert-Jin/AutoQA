# Midscene 与 Open-AutoGLM 项目学习笔记

> 记录日期：2026-03-14

## 1. Midscene.js 概览

**定位**：基于 VLM 的多平台 UI 自动化 SDK（TypeScript/Node.js）

### 核心架构

- **分层设计**：
  - Layer 0: `@midscene/shared` — 公共工具、类型、图像处理
  - Layer 1: `@midscene/core` — Agent 引擎、AI 模型、规划
  - Layer 2: 平台适配层 — web/android/ios/computer/harmony
  - Layer 3: 上层应用 — MCP Server、CLI、Chrome 插件、Playground

- **核心流程 Plan → Locate → Act**：
  1. 截图 + 用户指令 → VLM 规划出动作序列
  2. 每个动作通过 VLM 定位元素坐标
  3. 在目标设备执行操作

- **三种规划策略**：
  | 策略 | 特点 | 重规划上限 |
  |------|------|-----------|
  | 通用 LLM | 一次输出多步，需二次 AI Locate | 20 |
  | UI-TARS | 单步，像素坐标直出，跳过 Locate | 40 |
  | AutoGLM | 单步，[0,1000] 归一化坐标，跳过 Locate | 100 |

- **设备抽象**：`AbstractInterface` 统一所有平台的截图、操作接口

- **特色功能**：YAML 脚本系统、缓存机制、HTML 报告生成、MCP 协议支持

### 关键设计模式

- 策略模式（规划算法切换）
- 适配器模式（设备抽象）
- 工厂模式（Action 定义）
- 模板方法（MCP 工具生成）

## 2. Open-AutoGLM 概览

**定位**：智谱开源的手机端 Agent 框架（Python）

### 核心架构

- **流程**：截图 → AutoGLM 视觉模型 → 输出动作（归一化坐标）→ ADB/HDC 执行 → 循环
- **代码结构**：
  ```
  phone_agent/
  ├── agent.py          # PhoneAgent 主类
  ├── adb/              # ADB 设备控制
  ├── hdc/              # HDC（鸿蒙）设备控制
  ├── actions/handler.py # 动作执行器
  ├── config/           # 配置（App 映射、中英文 Prompt）
  └── model/client.py   # OpenAI 兼容 API 客户端
  ```

- **模型**：AutoGLM-Phone-9B（中文优化）和 Multilingual 版本
- **坐标系**：[0, 1000] 归一化坐标
- **支持操作**：Launch / Tap / Type / Swipe / Long Press / Double Tap / Back / Home / Wait / finish
- **设备支持**：Android（ADB）、鸿蒙（HDC），支持远程 WiFi 调试
- **模型部署**：可用智谱 BigModel API、ModelScope，或本地 vLLM/SGLang

### 接口层级

| 层级 | 组件 | 能力 |
|------|------|------|
| 顶层 | `PhoneAgent.run()` / `.step()` | 完整 Agent 循环 |
| 中层 | `ModelClient` / `ActionHandler` | 模型调用 + 动作执行 |
| 底层 | `DeviceFactory` (ADB/HDC) | 设备操作 |

## 3. 两个项目对比

| 维度 | Midscene.js | Open-AutoGLM |
|------|------------|--------------|
| 语言 | TypeScript/Node.js | Python |
| 平台 | Web + Android + iOS + 桌面 + 鸿蒙 | Android + 鸿蒙 (+ iOS 实验性) |
| 模型 | 支持多种 VLM（GPT-4o、Claude、Qwen、Gemini 等） | AutoGLM-Phone-9B 专用模型 |
| 操作与断言 | 同一模型既操作又断言 | 仅操作，无内置断言 |
| 坐标 | LLM 文本描述 / UI-TARS 像素 / AutoGLM 归一化 | [0, 1000] 归一化 |
| 规划 | 支持多步 + 单步 | 仅单步（每轮一个动作） |
| 脚本 | YAML 脚本系统 | 命令行 / Python API |
| 缓存 | 有（plan + locate） | 无 |
| 报告 | HTML 报告 + 可视化回放 | 无（verbose 日志输出） |
| MCP | 完整 MCP Server 支持 | 无 |
| 定位 | 二次 AI Locate（LLM 模式） | 模型直出坐标，无需二次定位 |

## 4. AutoQA 框架的设计理念

基于以上两个项目的分析，AutoQA 的核心创新点：

### 双模型分离（与 Midscene 最大区别）

- **AutoGLM（手）**：专门训练过手机操作，动作精准
- **VLM（眼）**：Qwen3-VL / Gemini 2.5 Pro，做语义级视觉断言

### 复用策略

- 复用 Open-AutoGLM 的中底层组件（ModelClient、ActionHandler、DeviceFactory）
- 不使用 PhoneAgent 的顶层 run()/step()，因为：
  - `run()` 一次跑完整个任务，无法中途插入断言
  - `step()` 后续轮次传不了新指令

### 新增组件

- TestRunner（测试编排）
- TestExecutor（上下文管理）
- Asserter（视觉断言 + 可插拔 VLM Provider）
- ReportGenerator（可视化报告）

详见 `AutoQA-ARCHITECTURE.md` 和 `AutoQA-CONTEXT.md`。
